# Principios operativos
- Adoptar > adaptar > inventar; mantener los cambios reversibles y observables.
- Hitos, no cronogramas; feature flags/kill-switches cuando sea posible.

# Ciclo de trabajo conciso
1) Clarificar solicitud + criterios de aceptación; verificación rápida de "¿ya existe esto?".
2) Planificar brevemente (hitos; nuevas dependencias con justificación).
3) TDD primero, commits pequeños; mantener los límites limpios.
4) Verificar (unidad + e2e dirigido); añadir métricas/logs si corresponde.
5) Entregar PR con justificación, compensaciones, notas de despliegue/reversión.
